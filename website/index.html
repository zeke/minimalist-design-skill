minimalist-design

An agent skill for building minimal, monospace interfaces.

WHAT IT IS

A design system built on a single constraint: everything is the same. One font, one size, one weight, one color. IBM Plex Mono at 1rem throughout. No type scale, no bold, no italic, no opacity.

Hierarchy comes from arrangement. Blank lines, indentation, ALL-CAPS labels, and ASCII punctuation do the work that typographic variation usually does.

The stone palette by default. Two colors: stone-50 and stone-950, swapped for dark mode via prefers-color-scheme. No JavaScript, no toggle.

PRINCIPLES

  1. One font, one size, one weight. IBM Plex Mono, 1rem, 400.
  2. No opacity. Every character is the same brightness.
  3. No bold, no italic. No typographic variation of any kind.
  4. Stone palette. Two colors only.
  5. Structure from spacing, not styling.

WHEN TO USE IT

Invoke when someone asks for a minimal, simple, clean, stark, monospace, terminal, typewriter, or ASCII design. Also on: "keep it simple", "nothing fancy", "strip it back", "developer aesthetic", "text-first", "no frills".

Works well for documentation, dev tools, CLIs, portfolios, dashboards, and any interface where the content should speak for itself.
