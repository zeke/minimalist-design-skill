minimalist-design

An agent skill for building minimal, monospace interfaces.

What it is

A design system that uses a single monospace font at a single size. IBM Plex Mono throughout, 1rem everywhere. No type scale. Hierarchy comes from font weight and opacity, not size.

The stone palette by default — a warm neutral gray that works in light and dark mode. Dark mode is handled entirely by prefers-color-scheme. No JavaScript, no toggle.

No shadows, no gradients, no decorative borders. Whitespace carries the structure.

Principles

  1. One font, one size. IBM Plex Mono at 1rem everywhere.
  2. Opacity creates hierarchy — 100% primary, 70% secondary. Both pass WCAG AAA.
  3. Stone palette by default. Swap for zinc, slate, or neutral.
  4. No shadows, gradients, or decorative borders.
  5. System dark mode. No JavaScript.

When to use it

Invoke this skill when someone asks for a minimal, simple, clean, stark, or monospace design. Also when they say things like:

  • keep it simple
  • nothing fancy
  • just make it clean
  • strip it back
  • developer aesthetic
  • text-first
  • no frills

Works well for documentation, dev tools, CLIs, portfolios, dashboards, and landing pages where clarity matters more than decoration.
