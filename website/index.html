A minimalist design system for AI-generated interfaces. IBM Plex Mono throughout, monochromatic stone palette, opacity-based hierarchy, dark and light mode.

npx skills add zeke/minimalist-design-skill

Five principles

01

One font, one size

IBM Plex Mono at 1rem everywhere. No type scale. Weight and opacity carry hierarchy, not size.

02

Opacity, not hue

Secondary text at 60%, tertiary at 35%. Never change the stone hue to de-emphasize content.

03

Stone by default

Warm neutral gray throughout. Swap for zinc, slate, or neutral if the palette doesn't fit the project.

04

No decoration

No shadows, no gradients, no large rounded corners. Borders only when they carry meaning. Whitespace is the structure.

05

System dark mode

Honors prefers-color-scheme with no JavaScript. Every color token has a dark counterpart.

"The best design is invisible. You notice the content, not the container."

Typography

One size. Weight and opacity do all the work.

font-medium — headings, labels, emphasis

font-normal — body text, default for most content

opacity-60 — secondary text, descriptions, metadata

opacity-35 — hints, placeholders, tertiary content

italic opacity-60 — captions, asides, annotations

Code

Same font everywhere means code blocks feel native, not foreign.

// one font, one size, everywhere
* {
  font-family: 'IBM Plex Mono', ui-monospace, monospace;
  font-size: 1rem;
  line-height: 1.6;
}

Form

Inputs with border focus, no rounded corners, no shadows.

optional context for your message

Callouts

Left border accent instead of color backgrounds.

Note

Use border-l-2 at full opacity to call out neutral information without introducing color.

Warning

Amber is the one semantic exception. Use only for genuine warnings, not decoration.

Error

Red for errors. The only other semantic exception alongside amber.

npx skills add zeke/minimalist-design-skill
